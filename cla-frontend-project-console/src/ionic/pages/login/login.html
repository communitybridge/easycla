CLA Project Management Console

Making CLAs Easy for Maintainers

Reduce administrative hassles of supporting the CLA for your project

  • Support both individual and corporate CLAs
  • Look in one place for companies and individuals who have signed the CLA
  • Consolidate all your signed CLA in a single portal
  • Push whitelisting requirements to the corporate contributors
  • Reduce likelihood of errors during enforcement of CLA signing

How to Start Using EasyCLA as a Project Manager

Step 1

Setup project on EasyCLA
  • All Linux Foundation projects on Github or Gerrit can use EasyCLA
  • Email support@linuxfoundation.org to confirm whether your specific project has been set up on EasyCLA
  • If your project is not on Linux Foundation project, request access by filling out this form [HERE]
  • We evaluate non-Linux Foundation projects on a case by case basis

Step 2

Get your Linux Foundation ID
  • When your project was given access to EasyCLA, you should have been given a Linux Foundation (LF ID) to log in.

    • If you do not have your LF ID, email support@linuxfoundation.org to request it.

Step 3

Login to Manage CLAs
 Once you are able to login you will be able to do the following to simplify the CLA process for your project.
  • Add Repos
  • Request your specific CLA
  • Review signatory Companies and Individuals
